# QuickDict 使用说明

## 简介

QuickDict（QECDict）是一款基于 ECDICT 词典数据库的 Windows 屏幕取词翻译工具。
支持 UI Automation + OCR 双通道取词，适用于浏览器、编辑器、PDF、图片、投屏等场景。

词典数据来源：ECDICT（340 万词条）
项目主页：https://github.com/skywind3000/ECDICT

## 部署步骤

1. 解压 QuickDict 程序包到任意目录。
2. 解压 QuickDict-data 数据包。
3. 将 QuickDict-data/data/ 复制到 QuickDict/ 下(release包不包含，构建release会生成。)。

### 自行生成数据库（不使用 QuickDict-data 包）

如果你没有单独的数据库包，也可以自己生成 `ecdict.db`：

1. 准备完整版词典源文件：`stardict/stardict.csv`。
2. 在项目根目录运行 `setup.bat`（会自动创建虚拟环境、安装依赖并构建数据库）。
3. 构建完成后，确认 `data/ecdict.db` 已生成。
4. 将 `data/ecdict.db` 复制到发布目录的 `QuickDict/data/` 下。

说明：数据库体积较大（约 800MB）

最终目录结构：

```text
QuickDict/
  QuickDict.exe
  使用说明.md
  _internal/
  data/
    ecdict.db
```

4. 双击 QuickDict.exe 启动。

## 快捷键

- `Ctrl x 2`：激活 / 关闭取词模式
- `Esc`：退出取词模式
- Ctrl 触发方式下，激活后单按一次 `Ctrl`：对当前鼠标位置取词

## 触发方式（托盘 -> 触发方式）

- 悬停取词：鼠标悬停自动取词
- Ctrl 键取词：仅在单按 Ctrl 时取词

## 取词模式（托盘 -> 取词模式）

- 自动（UIA -> OCR）：优先 UIA，失败回退 OCR（推荐）
- 仅 UIA：适合标准文本控件
- 仅 OCR：适合图片、PDF、视频、投屏

## 新增能力（已集成）

- OCR 多策略预处理：CLAHE、Otsu、自适应阈值、形态学去线
- 低分辨率增强：2x 放大 + 锐化/增强变体
- Teams/硬件加速窗口支持：GDI 黑屏时自动回退 `dxcam`
- 多显示器坐标修复：支持副屏与负坐标
- 选词精度优化：OCR 结果按同行优先（Y 方向加权）
- 截图区域可视化：托盘开关可显示 OCR 截图框
- 取词状态指示器：托盘开关，取词开启时常驻显示 `取词中`
- 菜单配置持久化：截图区域和状态指示器开关会写入设置文件

## 常用托盘菜单

- 开启/关闭取词
- 取词模式（自动 / 仅 UIA / 仅 OCR）
- 触发方式（悬停 / Ctrl）
- 显示截图区域（调试）
- 状态指示器（仅显示 `取词中`）
- 退出

## 注意事项

1. 程序常驻系统托盘，无主窗口。
2. `_internal` 为运行依赖目录，请勿删除。
3. OCR 首次使用会加载模型，可能有 1-2 秒延迟。
4. 专业术语或非常规字体可能无法命中词典。

## 常见问题

### 为什么某些窗口 OCR 失败？

已内置 GDI -> dxcam 回退。若仍失败，建议切换为 `仅 OCR` 并开启截图区域观察目标是否在框内。

### 为什么识别到鼠标上方文字？

已优化为同行优先。若仍偶发，建议缩小目标区域（稳定悬停）并开启截图区域辅助定位。

### 如何完全退出？

托盘右键 -> 退出。
