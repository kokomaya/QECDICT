# Magic Mirror — 界面与交互优化方案

> 基于当前代码的全面 UI/UX 审查，按优先级分为 **P0 (必须)** / **P1 (重要)** / **P2 (锦上添花)** 三档。  
> 每条建议包含：现状问题 → 优化目标 → 实现要点。

---

## P0 — 产品级必须修复

### 1. 翻译失败无用户反馈

- **现状**：`_on_pipeline_error` 仅 `logger.error()`，用户完全不知道翻译失败了，覆盖层停留在骨架屏状态
- **优化**：
  - 骨架屏区域显示红色错误提示条 "翻译失败，右键重试"
  - 同时发送系统托盘气泡通知（与 OCR 失败一致）
  - 覆盖层右键菜单保留"重新翻译"入口

### 2. 框选时无尺寸/坐标反馈

- **现状**：`RegionSelector` 框选过程中仅显示蓝色边框 + 镂空区域，无任何数值信息
- **优化**：
  - 框选边框附近实时显示 **宽×高 像素** 尺寸标注（如 `420 × 180`）
  - 可选：显示鼠标当前坐标 → 方便精确框选

### 3. 系统托盘缺少图标

- **现状**：`_setup_tray()` 中 `QSystemTrayIcon` 未调用 `setIcon()`，Windows 托盘区显示空白/默认图标
- **优化**：
  - 设计一个 Magic Mirror 品牌图标（32×32 ICO/PNG）
  - 在 `_setup_tray()` 中 `self._tray.setIcon(QIcon("path/to/icon.png"))`

### 4. 聊天窗口无法复制 AI 回复

- **现状**：`ChatHtmlView` 设置了 `NoContextMenu`，QWebEngineView 内文本无法选中复制
- **优化**：
  - 恢复 `DefaultContextMenu` 或实现自定义右键菜单（复制选中 / 复制全部）
  - 代码块右上角添加 **一键复制按钮**（通过 HTML/JS 注入 clipboard API）
  - AI 消息块右上角添加 **复制整条回复** 按钮

### 5. 左键穿透未真正实现

- **现状**：`mirror_overlay.py` 注释写着 "左键穿透通过 nativeEvent + WM_NCHITTEST 实现"，但代码中只有 `event.ignore()`，并无 `nativeEvent` 实现 → 左键实际仍被覆盖层拦截
- **优化**：
  - 实现 `nativeEvent` 拦截 `WM_NCHITTEST`，对左键返回 `HTTRANSPARENT`
  - 或使用 `WS_EX_TRANSPARENT` 扩展窗口样式（需平衡右键菜单可用性）

## P1 — 显著提升用户体验

### 6. 骨架屏脉冲动画

- **现状**：骨架占位条内有一个固定宽度的深色条纹，模拟"脉冲"，但实际是 **静态的**，视觉上像渲染 Bug 而非加载状态
- **优化**：
  - 使用 `QVariantAnimation` 让深色条从左到右循环滑动（shimmer 效果）
  - 参考 Material Design skeleton loading 标准动画

### 7. 覆盖层可拖拽移动

- **现状**：覆盖层固定在框选位置，遮挡了需要参照的内容时只能关闭重来
- **优化**：
  - 添加一个 **小拖拽手柄**（覆盖层顶部/角落），按住可移动覆盖层
  - 或：按住 Alt + 左键拖拽移动
  - 同步移动关联的 `ContextPreviewPanel`

### 8. 覆盖层可缩放/调整大小

- **现状**：覆盖层大小完全由框选区域决定，翻译文字溢出时只能字号缩小
- **优化**：
  - 覆盖层边缘添加 resize 手柄（右下角或四边）
  - 调整大小后重新计算字号排版
  - 或：双击覆盖层自动扩展到适合内容的大小

### 9. 快捷键可自定义

- **现状**：`Ctrl+Alt+T` / `Ctrl+Alt+C` 硬编码在 `settings.py`，可能与用户已有快捷键冲突
- **优化**：
  - 在系统托盘菜单增加"设置"入口 → 打开设置对话框
  - 提供热键录制控件（按下组合键自动识别并保存）
  - 热键配置持久化到 `data/settings.json`

### 10. 设置界面

- **现状**：所有配置参数散落在 `settings.py` 常量中，用户无法修改（`data/settings.json` 存在但未对接 `config/settings.py`）
- **优化**：
  - 设计一个设置对话框（`SettingsDialog`），分为：
    - **快捷键**：翻译 / 提取 / 关闭全部
    - **外观**：字体、字号缩放、覆盖层透明度、主题（暗色/浅色）
    - **翻译**：默认目标语言、LLM Provider 选择
    - **高级**：OCR 置信度阈值、GPU 开关
  - 读写 `data/settings.json`，应用启动时加载

### 11. 翻译进度指示

- **现状**：骨架屏显示后，用户无法知道总共几个文本块、已翻译几个
- **优化**：
  - 在 `ContextPreviewPanel` 顶部或 `LoadingIndicator` 上显示进度：`翻译中 3/8`
  - 或在覆盖层底部显示细长进度条

### 12. 翻译取消机制

- **现状**：启动翻译后无法中途取消，必须等待全部完成或超时
- **优化**：
  - `LoadingIndicator` 添加 "取消" 按钮或提示 "按 Esc 取消"
  - Esc 键在翻译进行中的语义从"关闭覆盖层"变为"取消当前翻译"
  - 通过 threading Event 或 cancel flag 中止 `_StreamingPipelineWorker`

### 13. 聊天窗口：新建对话 / 清空历史

- **现状**：`ChatDialog` 有 `_clear()` 方法但 UI 未暴露入口；多次打开智能对话会创建独立窗口，无法延续上下文
- **优化**：
  - 顶栏添加"清空对话"按钮（🗑 图标）
  - 考虑单例聊天窗口 + 侧边栏历史会话列表
  - 或：允许从不同覆盖层追加上下文到同一对话

### 14. 首次启动引导（Onboarding）

- **现状**：用户首次启动只看到系统托盘图标（可能还是空白的），不知道如何操作
- **优化**：
  - 首次启动显示一个欢迎窗口：
    - 展示核心快捷键（翻译 / 提取 / 关闭）
    - 检查 LLM 配置是否就绪，缺失则引导配置
    - "开始使用"按钮直接进入框选翻译
  - 后续启动显示托盘气泡提示 "Magic Mirror 已就绪，按 Ctrl+Alt+T 开始翻译"

## P2 — 竞品差异化 & 细节打磨

### 15. 多语言支持

- **现状**：界面文本全部硬编码中文（"翻译中..."、"复制全部译文"等），翻译方向固定为 EN→ZH
- **优化**：
  - UI 文本抽取为 i18n 资源文件
  - 支持翻译方向配置（源语言 → 目标语言）
  - 自动检测源语言（利用 OCR 结果判断）

### 16. 翻译历史记录

- **现状**：关闭覆盖层后翻译内容彻底消失，无法回看
- **优化**：
  - 每次翻译自动保存记录（时间戳 + 原文 + 译文 + 截图缩略图）
  - 系统托盘菜单增加"翻译历史"入口
  - 历史窗口支持搜索、复制、重新翻译

### 17. 高 DPI / 多显示器缩放适配

- **现状**：未主动处理 DPI 缩放，依赖 Qt 默认行为；不同缩放比例的多显示器场景可能错位
- **优化**：
  - 在 `main.py` 启动时显式设置 `QApplication.setHighDpiScaleFactorRoundingPolicy`
  - 框选坐标和截图坐标统一使用物理像素
  - 覆盖层定位和字号计算纳入 DPI 比例因子

### 18. 浅色主题 & 自适应配色

- **现状**：所有 UI 组件强制深色主题，ChatDialog / ContextPreviewPanel / LoadingIndicator 颜色硬编码
- **优化**：
  - 检测系统暗色/亮色模式偏好（`QApplication.styleHints().colorScheme()`）
  - 提供浅色 / 深色 / 跟随系统三种选项
  - `chat_theme.py` 重构为 Theme 类，支持运行时切换

### 19. 框选区域微调

- **现状**：框选完成后无法微调边界，框选不精确只能重来
- **优化**：
  - 框选完成后短暂进入"确认模式"：
    - 四角/四边显示调整手柄，可拖拽微调
    - Enter 确认开始翻译，Esc 取消
  - 或：按住 Shift 键时以 1px 步进精确移动选区边缘

### 20. 动效与过渡优化

- **现状**：
  - 覆盖层直接 `show()`，无出现过渡
  - 关闭覆盖层直接 `hide()`，无消失动画
  - `ContextPreviewPanel` 直接显示/隐藏
- **优化**：
  - 覆盖层出现时从 0% 淡入到 100%（200ms）
  - 关闭时淡出（150ms）后再 `hide()`
  - `ContextPreviewPanel` 从右侧滑入（slide-in）
  - 所有动画时长可配置

### 21. 全局快捷悬浮球

- **现状**：触发翻译必须记住快捷键或找到系统托盘
- **优化**：
  - 可选的半透明悬浮球（类似手机辅助触控），停靠在屏幕边缘
  - 点击展开：翻译 / 提取 / 对话 / 设置
  - 可拖拽，双击隐藏
  - 在设置中开关

### 22. OCR 识别区域高亮反馈

- **现状**：OCR 完成后直接显示骨架屏，用户不知道识别了哪些文本区域
- **优化**：
  - OCR 完成后短暂（300ms）高亮显示检测到的文本块边界框（半透明蓝色描边）
  - 然后过渡为骨架屏占位条
  - 帮助用户判断 OCR 是否准确捕捉到了目标文本

### 23. 跨平台字体回退链

- **现状**：`FONT_FAMILY_ZH = "Microsoft YaHei"` 仅适用 Windows；`md_renderer.py` 字体栈也是 Windows 特定
- **优化**：
  - 统一字体回退链：`"Microsoft YaHei", "PingFang SC", "Noto Sans CJK SC", "WenQuanYi Micro Hei", sans-serif`
  - 或在 `settings.py` 中按 `sys.platform` 自动选择

### 24. 覆盖层边缘指示器

- **现状**：覆盖层完全无边框，与屏幕原始内容融合，用户可能分不清哪些区域是覆盖层
- **优化**：
  - 覆盖层四角或边缘显示细微的半透明圆角边框/角标
  - hover 时显示更明显的边界提示
  - 帮助用户定位覆盖层范围

### 25. 截图预览对比

- **现状**：翻译覆盖原文后，用户无法快速对比原文和译文
- **优化**：
  - 长按覆盖层（或按住某个快捷键如 Space）临时隐藏覆盖层，松开恢复 → "透视"原文
  - 或在 `ContextPreviewPanel` 中同时展示原文和译文的对照
